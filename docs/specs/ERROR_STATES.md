# Error States & Edge Cases Specification

## Overview

This document defines how Plinth handles errors, edge cases, and unusual states. Consistent error handling improves user trust and debugging.

---

## Error Handling Principles

1. **Be specific** - Tell users what went wrong, not just "Something went wrong"
2. **Be actionable** - Always provide a next step
3. **Be honest** - Don't hide errors or pretend everything is fine
4. **Fail gracefully** - Partial functionality is better than none
5. **Log everything** - Errors should be traceable for debugging

---

## Error Logging & Monitoring

### Client-Side Logging

```typescript
// Sentry integration
Sentry.init({
  dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
  environment: process.env.NODE_ENV,
  tracesSampleRate: 0.1,
});

// Custom error boundary
function logError(error: Error, context: object) {
  Sentry.withScope((scope) => {
    scope.setExtras(context);
    Sentry.captureException(error);
  });
}
```

### Error Context

Always include:
- User ID (anonymized if needed)
- Organization ID
- Current page/route
- Action being attempted
- Request/response data (sanitized)

### Alert Thresholds

| Error Type | Alert Threshold |
|------------|-----------------|
| 5xx errors | >5 in 5 minutes |
| AI failures | >10% of requests |
| Auth errors | >20 in 10 minutes |
| Payment errors | Any |

## Recovery Strategies

### Autosave with Recovery

```typescript
// Save drafts to localStorage every 30s
const autosave = {
  interval: 30000,
  key: `plinth_draft_${decisionId}`,
  maxAge: 7 * 24 * 60 * 60 * 1000, // 7 days
};

// On page load, check for recovery
if (hasLocalDraft(decisionId)) {
  showRecoveryPrompt({
    title: "Recover unsaved changes?",
    message: "You have unsaved changes from your last session.",
    actions: [
      { label: "Recover", action: () => applyDraft() },
      { label: "Discard", action: () => discardDraft() }
    ]
  });
}
```

### Retry Logic

```typescript
const retryConfig = {
  maxRetries: 3,
  backoff: 'exponential',
  baseDelay: 1000,
```

### Graceful Degradation

When AI services fail:
- Allow manual entry of options/evidence
- Show cached competitor profiles (with stale warning)
- Generate simpler brief using templates instead of AI
